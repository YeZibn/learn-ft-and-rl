# Module 2: Transformer and SFT Foundations

## 2.1 模块目标

学完本模块，你应该能回答：

1. Transformer 为什么能作为大模型底座
2. decoder-only 模型到底在预测什么
3. 自回归语言建模训练目标是什么
4. SFT 为什么有效
5. SFT 为什么又不够
6. 为什么 Agent 训练不能只看 loss
7. 为什么数据质量比单纯堆量更重要

## 2.2 Transformer 的角色

Transformer 是当前大语言模型的核心结构。  
在 decoder-only 范式下，它的关键任务是根据前面的 token 预测下一个 token。

这件事看似简单，但当模型规模、数据规模和训练质量足够高时，它会学出：

- 语言
- 知识
- 风格
- 格式
- 指令响应
- 某些推理模式

## 2.3 Decoder-only 架构

Decoder-only 模型是从左到右生成文本的。

例如：

```text
User: 请总结这段话
Assistant:
```

模型要做的是继续生成 assistant 的回答。

### 关键直觉

模型不是一次性生成整段答案，而是一个 token 一个 token 地往外生成。

这会直接影响：

1. 训练目标
2. 输出稳定性
3. 结构化格式控制
4. 失败时的行为

## 2.4 自回归语言建模目标

大模型最基础的训练目标就是：

> 给定前面的 token，预测下一个 token

这通常对应 token-level cross entropy loss。

### 这个目标学到的是什么

1. 语言连续性
2. 常见表达方式
3. 任务格式
4. 隐式推理模板
5. 大量统计规律

### 这个目标没直接保证什么

1. 回答一定正确
2. 工具调用一定稳定
3. 多步任务一定完成
4. Agent 行为一定可靠

## 2.5 为什么 token-level loss 不等于任务能力

loss 下降，只说明模型更会预测训练数据里的 token。  
它不自动保证：

- 回答更正确
- 工具调用更稳定
- 多步任务更能完成
- Agent 行为更可靠

### 一个直觉例子

模型可以学会：

- 说得更像样
- 格式更工整
- 术语更熟练

但未必真的会：

- 找对工具
- 处理异常
- 正确停下
- 在任务失败后修正策略

## 2.6 什么是 SFT

SFT = Supervised Fine-Tuning，监督微调。

它的本质是：

> 用人工整理好的输入输出样本，教模型模仿标准行为。

SFT 让模型学习：

- 看到这种输入时应该怎么输出
- 应该遵守什么格式
- 应该呈现什么对话风格
- 工具调用长什么样

## 2.7 SFT 解决什么问题

SFT 特别适合解决：

1. 指令跟随
2. 对话风格统一
3. 格式约束
4. 工具调用模板学习
5. 多轮对话结构
6. 领域风格适配

### 优点

1. 直接
2. 稳定
3. 可控
4. 易落地

## 2.8 SFT 不擅长什么

SFT 不擅长直接解决：

1. 稀疏奖励问题
2. 复杂多步决策优化
3. 偏好排序问题
4. 环境交互中的探索问题
5. 长链条任务的全局最优

原因很简单：SFT 只是在模仿样本，不是在主动优化结果。

## 2.9 instruction tuning 是什么

instruction tuning 可以理解成 SFT 的一种重要形式。  
它强调让模型学会按自然语言指令工作。

例如：

- 请总结
- 请翻译
- 请调用工具查询
- 请按 JSON 输出

它和普通语言建模的差别在于：

1. 更强调任务行为
2. 更强调对话结构
3. 更强调按要求行动

## 2.10 为什么 SFT 会有效

SFT 有效，是因为它把抽象目标变成了可模仿样本。

模型不需要自己先理解“如何做人类助手”，它只需要从大量示范中学出：

- 什么时候该回答
- 怎么组织语言
- 什么风格更符合预期
- 工具调用长什么样

这就是为什么高质量数据特别重要。

## 2.11 为什么 SFT 又不够

SFT 的局限很明显：

1. 只能学到样本里出现过的行为
2. 很难直接优化结果指标
3. 对偏好差异的处理不够细
4. 很难处理长链条任务的全局优化
5. 无法天然解决“做得对但不够好”的问题

所以 SFT 通常是第一步，不是终点。

## 2.12 训练退化是什么

训练退化通常指：

> 模型在某些方面变强了，但在另一些方面变差了。

例如：

- 指令更听话了，但知识回答变弱了
- 格式更工整了，但开放问答变差了
- 工具调用更频繁了，但滥用工具了

### 常见原因

1. 数据分布偏了
2. 学习率太大
3. 训练过头
4. 模板错配
5. 任务混合比例不合理

## 2.13 数据质量为什么更重要

模型训练不是“数据越多越好”，而是“数据越对越好”。

对 SFT 来说，数据质量至少包括：

1. 正确性
2. 一致性
3. 格式规范
4. 覆盖面
5. 难度分布
6. 行为结构清晰度

### 对 Agent 尤其重要

因为 Agent 学的不是单轮答案，而是行为轨迹。  
如果轨迹本身不清晰，模型学不到正确决策链。

## 2.14 一个更实用的直觉

SFT 在优化的是：

> 让模型在给定上下文下，更倾向生成训练样本里的正确后续 token

它不是直接优化任务成功，而是通过模仿成功样本来逼近任务成功。

因此：

- 好样本很值钱
- 坏样本很危险
- 模板一致性很重要

## 2.16 模块 2 的核心结论

1. Transformer 的核心是 next-token prediction
2. SFT 本质是模仿学习，不是结果优化
3. loss 下降不等于任务能力提升
4. 数据质量决定 SFT 上限
5. Agent 训练要看行为和任务，而不是只看文本
