# Module 1: Qwen Model Ecosystem

## 1.1 模块目标

理解 Qwen 系列模型、Tokenizer、Chat template、工具调用格式和训练生态。

学完本模块，你应该能回答：

1. Qwen 有哪些常见形态
2. Base 和 Instruct 的区别是什么
3. tokenizer 为什么重要
4. chat template 为什么是协议而不是装饰
5. tool calling 为什么是 Agent 的关键接口
6. 训练和推理分别该用哪些工具

## 1.2 总体结构

Qwen 生态可以理解成四层：

1. 模型本体
2. 输入输出协议
3. 训练与微调生态
4. 推理与部署生态

后面的 Agent、SFT、偏好学习、RL，都要建立在这四层之上。

## 1.3 Qwen 模型家族怎么理解

### Base 和 Instruct

#### Base

Base 模型主要做语言建模，核心能力是：

- 续写
- 补全
- 学习语料分布

它不天然擅长：

- 指令跟随
- 多轮对话
- 工具调用
- 结构化输出

#### Instruct

Instruct 模型是在 Base 上做过指令微调后的版本，更适合：

- 对话
- 助手
- 工具调用
- Agent 原型

#### 你要记住

- 做研究和继续训练时，Base 更常见
- 做应用和 Agent 原型时，Instruct 更省事
- 选择哪一个，不是好坏问题，而是用途问题

### Dense 和 MoE

#### Dense

所有参数基本都会参与前向计算。

优点：

- 结构简单
- 训练稳定
- 便于调试

#### MoE

Mixture of Experts，按路由激活部分参数。

优点：

- 容量大
- 推理更省
- 扩展性好

缺点：

- 训练和部署更复杂
- 路由行为更难解释

### 多模态版本

Qwen 也有多模态方向，支持文本之外的图像等输入。

对 Agent 来说，这很重要，因为很多任务不是纯文本，而是：

- 看截图
- 读文档
- 理解页面
- 处理图片内容

但在这条路线里，先把文本底座吃透，再扩展到多模态。

## 1.4 tokenizer 为什么重要

tokenizer 负责把文本切成模型可处理的 token 序列。

模型不是直接看汉字或英文单词，而是看 token。

### 为什么这件事重要

因为下面这些都受 tokenizer 影响：

1. 上下文长度
2. 训练和推理成本
3. JSON 和代码的稳定性
4. stop token 的控制
5. tool call 输出的可解析性

### 你要特别注意

1. 同一句话，不同 tokenizer 切法不同
2. 训练和推理必须用同一个 tokenizer
3. 对结构化输出，token 切分会直接影响稳定性

## 1.5 chat template 是什么

chat template 是把多轮消息拼成模型输入格式的规则。

典型角色包括：

- system
- user
- assistant
- tool

### 它为什么重要

因为模型训练时学到的不是“随便拼接的字符串”，而是固定协议下的输入输出习惯。

如果你推理时的模板和训练时不一致，模型表现会明显变差。

### 常见错误

1. system prompt 放错位置
2. assistant 标记缺失
3. tool message 格式不统一
4. 训练和推理模板不一致
5. stop token 设置错误

### 关键理解

chat template 不是展示格式，而是训练和推理协议。

## 1.6 tool calling 为什么是关键能力

Agent 的核心不是“会说工具名”，而是“会完成动作闭环”。

一个合格的 tool call 至少包含三件事：

1. 选对工具
2. 填对参数
3. 接住返回结果并继续推理

### 常见问题

1. 工具名写对了，参数错了
2. 参数格式对了，语义错了
3. 只会调用，不会根据结果继续
4. 工具报错后不会恢复
5. 遇到异常就胡编

所以工具调用训练不能只教“动作”，还要教“动作之后怎么处理”。

## 1.7 训练生态

### Transformers

最基础的加载、推理和训练框架。

适合：

- 模型验证
- 训练原型
- 推理实验

### PEFT

参数高效微调框架，常用于：

- LoRA
- QLoRA

适合：

- 显存有限时训练
- 快速验证
- 小规模实验

### TRL

更偏偏好学习和 RL。

适合：

- DPO
- PPO 类训练
- 对齐实验

### LLaMA-Factory / ms-swift / verl

更偏工程封装和训练流程整合。

适合：

- 快速跑通实验
- 减少胶水代码
- 复现实验流程

## 1.8 推理生态

训练不是终点，Agent 更关心部署后的表现。

推理生态要解决：

- 低延迟
- 高吞吐
- 长上下文
- 批处理
- 并发
- 结构化输出
- tool calling 服务化

常见推理框架包括：

- vLLM
- SGLang
- 其他高性能推理服务

对 Agent 来说，推理框架的重要性不低于训练框架。

## 1.9 什么时候用 Base，什么时候用 Instruct

### 用 Base 的场景

1. 继续预训练
2. 研究模型行为形成
3. 需要从更底层控制训练过程

### 用 Instruct 的场景

1. 直接做助手
2. 做工具调用原型
3. 做 Agent demo
4. 想快速看到效果

## 1.10 这门课中你要建立的判断

你要把下面这些判断变成直觉：

1. 模型不是系统，模型只是系统的一部分
2. chat template 不是装饰，而是协议
3. tool calling 不是输出 JSON，而是完成行为闭环
4. Base 和 Instruct 不是简单好坏，而是用途不同
5. 训练框架解决“怎么学”，推理框架解决“怎么跑”

## 1.11 模块 1 的核心结论

Qwen 生态的学习重点，不是记住一堆名词，而是理解：

1. 模型本体
2. 输入输出协议
3. 训练工具链
4. 推理部署链

这四层一旦搞清楚，后面你学 SFT、偏好学习、RL 和 Agent 才不会乱。
