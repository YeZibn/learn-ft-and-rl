# Module 0: Agent, Qwen, and Training Paradigms Overview

## 0.1 模块目标

建立全局地图，理解 Agent 训练不是单点算法问题，而是数据、模型、工具、奖励和评测共同作用的系统问题。

学完本模块，你应该能回答这几个问题：

1. 什么是 LLM Agent
2. Agent 和普通 Chat Model 的差别在哪里
3. Qwen 在整条学习路线中的位置是什么
4. 为什么训练通常要经历 `Continued Pretraining -> SFT -> Preference Learning -> RL`
5. Agent 的能力应该怎样拆解
6. 为什么评测必须比训练更早设计

## 0.2 学习前提

这个模块不要求你会做训练，但要求你对下面这些概念至少有感性理解：

1. 你知道什么是 Transformer
2. 你知道什么是 prompt 和 chat template
3. 你知道大模型是按 token 生成文本的
4. 你知道“训练”和“推理”是两件事
5. 你知道“模型输出好看”和“任务真的完成”不是一回事

如果这些还不稳，不影响继续学，但后面的内容会更容易断层。

## 0.3 这门课为什么以 Qwen 为中心

Qwen 在这条路线里承担的是“统一底座”的角色。

原因不是它唯一最好，而是它适合拿来做完整闭环：

1. 开源可用，方便本地复现
2. 指令能力和工具生态比较成熟
3. 适合做微调、偏好学习和 RL 练习
4. 容易接到真实 Agent 任务
5. 便于把训练、评测、部署串起来

因此，后面的内容不是“只讲 Qwen”，而是以 Qwen 为落点去理解整套方法论。

## 0.4 先建立一条主线

你可以先把整个课程压缩成一条主线：

`数据 -> 监督微调 -> 偏好学习 -> 强化学习 -> 评测闭环 -> 部署与迭代`

这条主线里，每一步解决的问题不同：

1. 数据决定模型能看到什么
2. SFT 决定模型会不会按样例做
3. 偏好学习决定模型更倾向哪种行为
4. RL 决定模型直接朝任务目标优化
5. 评测决定你看到的提升是不是假的
6. 部署和闭环决定模型能不能长期稳定工作

## 0.5 什么是 LLM Agent

LLM Agent 不是单纯的“会对话的模型”，而是一个能在环境中持续完成任务的系统。

最常见的拆法是四层：

### 1. Model

负责语言生成、决策候选、推理与控制输出格式。

### 2. Tools

负责查找、计算、执行和访问外部系统，例如：

- 搜索
- 数据库
- 日历
- 浏览器
- 代码执行器
- API

### 3. Memory

负责保存任务状态、长期偏好、历史记录和中间结论。

### 4. Controller

负责循环、规划、重试、终止条件、错误处理和多步调度。

一个真正的 Agent，通常不是“模型直接答完就结束”，而是：

1. 读任务
2. 判断是否需要工具
3. 选择动作
4. 观察环境反馈
5. 再规划
6. 重复直到完成

## 0.6 Agent 和普通 Chat Model 的差别

普通 Chat Model 的主要目标是“回答得像样”。

Agent 的主要目标是“把事做完”。

这带来几个很现实的差异：

### Chat Model 关注的指标

1. 回答是否自然
2. 是否符合指令
3. 是否安全
4. 是否信息完整

### Agent 关注的指标

1. 工具选得对不对
2. 参数传得对不对
3. 多步任务能不能做完
4. 出错后能不能修正
5. 最终任务是否完成

所以 Agent 训练不是“写得更像人”，而是“做得更像一个可靠执行系统”。

## 0.7 Agent 的核心难点

Agent 比普通对话难，主要因为 reward 的结构更差：

1. 延迟：最后才知道成没成
2. 稀疏：很多中间动作没有直接反馈
3. 噪声大：工具失败、环境异常、接口错误都会干扰
4. 可作弊：模型可能学会讨巧而不是真解决问题
5. 长链条：一个任务往往要很多步才能完成

这也是为什么在 Agent 里，单看 loss 基本不够，必须看任务完成率、工具成功率和恢复能力。

## 0.8 Agent 的能力拆解

为了训练和评测，通常把 Agent 能力拆成下面几块：

### 1. 指令跟随

模型能理解“你要它做什么”。

### 2. 工具使用

模型能知道“什么时候该用工具、用哪个工具、参数怎么传”。

### 3. 规划

模型能把大任务拆成小步骤。

### 4. 反思

模型能在失败后判断自己哪里错了。

### 5. 记忆

模型能记住任务过程和长期偏好。

### 6. 检索

模型能从外部知识源取回需要的信息。

### 7. 执行与恢复

模型能在出现错误时重试、换策略、补救。

这几个能力后面会分别对应不同的数据、训练和评测方式。

## 0.9 训练范式总览

这一部分非常重要，因为它决定你后面怎么看待各种方法。

### 1. Continued Pretraining

先在大规模语料上继续训练，让模型适应新领域语言分布。

适合解决：

- 领域词汇不足
- 文体不适应
- 新知识分布偏移

### 2. SFT

用监督样本教模型“应该怎么答、怎么做”。

适合解决：

- 指令跟随
- 格式输出
- 基础工具调用
- 多轮对话行为

### 3. Preference Learning

让模型学会“什么更好、什么更差”。

适合解决：

- 回答风格偏好
- 安全性偏好
- 多候选排序
- 行为倾向修正

### 4. RL

直接按奖励函数优化策略。

适合解决：

- 任务完成率
- 多步决策
- 稀疏奖励任务
- 环境交互任务

### 5. Evaluation

验证你到底有没有真的提升。

## 0.10 为什么常见路线是这条顺序

`Continued Pretraining -> SFT -> Preference Learning -> RL`

原因很简单：

1. 先让模型“懂这个领域”
2. 再让模型“会按样例做”
3. 再让模型“更偏向好行为”
4. 最后让模型“直接朝任务目标优化”

如果反过来，通常会出现两个问题：

- 模型基础不够，RL 很难稳定
- 没有清晰行为样本，偏好学习也不稳

## 0.11 SFT、偏好学习、RL 的关系

这三者经常被混在一起，但本质不同。

### SFT

是“示范学习”。

你给模型标准答案，它学会模仿。

### 偏好学习

是“排序学习”。

你给模型多个候选，它学会更偏向更好的那个。

### RL

是“目标优化”。

你不给唯一标准答案，而是给奖励，模型自己探索优化路径。

## 0.12 Agent 训练为什么更依赖数据

Agent 训练不是只调算法，数据往往更关键。

因为 Agent 的真实能力，往往不是“会不会说”，而是：

- 会不会选工具
- 会不会分解任务
- 会不会恢复失败
- 会不会在正确时机停下

这些东西很难靠少量手工规则调出来，必须让数据体现出正确行为结构。

## 0.13 为什么评测要先设计

如果没有评测，训练就没有目标。

对 Agent 来说，常见的假进步包括：

1. 文本更长了，但任务没更好
2. 解释更漂亮了，但工具用错了
3. 格式更工整了，但完成率没提升
4. 模型更自信了，但错误也更隐蔽了

所以评测要尽早明确：

- 你到底要优化什么
- 什么叫成功
- 什么叫失败
- 什么叫部分成功

## 0.14 课程中的评测对象

后面你会看到很多不同的指标，它们对应不同层次：

1. 文本层：loss、perplexity
2. 语义层：回答正确率
3. 行为层：工具调用准确率
4. 任务层：任务完成率
5. 系统层：稳定性、恢复率、延迟、成本

Agent 课真正重要的是后面三层。

## 0.15 课程中常见的三种错误理解

### 错误 1：loss 下降就代表 Agent 变强

不成立。loss 只是拟合程度，不等于任务能力。

### 错误 2：模型会说工具名就代表会用工具

不成立。会说不等于会传参，不等于会处理异常。

### 错误 3：RL 一定比 SFT 强

不成立。RL 只有在 reward 清晰、环境稳定、基础能力足够时才有效。

## 0.16 一个更实用的 Agent 思维模型

你可以把 Agent 看成一个闭环控制系统：

1. 输入任务
2. 生成计划
3. 选择动作
4. 观察反馈
5. 修正计划
6. 输出结果

训练的本质是让这个闭环越来越稳定、越来越高效、越来越少犯错。

## 0.17 模块 0 的结论

你后面所有学习都可以压成一句话：

> 先用 SFT 把“会做的样子”教出来，再用偏好学习和 RL 把“做得更好”压上去，最后用评测和数据闭环把它稳定住。
